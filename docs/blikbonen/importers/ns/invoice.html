blikbonen.importers.ns.invoice

An importer for NS invoice .pdf files. Download these from the 'Overview of payments' / 'Betaaloverzicht' tab.

This importer usually works, but since it is based on reading PDF files it is likely to fail at least some of the time. While blikbonen.importers.ns.travel_history.Importer can be used to add expenses as a liability, this importer can be used to collect these liabilities, add the subscription costs, and collect them to a single payment posting, for example a zerosum account.

See Config for the configuration options.

Example

The following example initializes the importer:

from blikbonen.importers.ns import invoice

invoice.Importer(
    {
        "main_account": "Assets:Zero-Sum-Accounts:PublicTransport:NS",
        "account_number": "12345678",
        "subscription_expense": "Expenses:PublicTransport:NS",
        "travel_expense": "Liabilities:PublicTransport:NS",
    }
)

115        rdr = etl.rename(rdr, 0, "narration")
116        rdr = etl.rename(rdr, 1, "startdate")
117        rdr = etl.rename(rdr, 3, "enddate")

120        rdr = etl.cutout(rdr, 2)

123        rdr = etl.convert(rdr, "amount", self._convert_amount)
124        rdr = etl.convert(rdr, "amount", lambda v: v.replace(",", "."))
125        # Filter out extra rows this way
126        rdr = etl.select(rdr, "amount", _is_number)
127        rdr = etl.select(rdr, lambda r: r["narration"] != "")
128        rdr = etl.addfield(
129            rdr,
130            "date",
131            lambda _r: datetime.datetime(
132                self.date(self.file).year,
133                self.date(self.file).month,
134                self.date(self.file).day,
135                tzinfo=pytz.timezone("Europe/Amsterdam"),
136            ),
137        )
138        rdr = etl.addfield(rdr, "memo", lambda r: f"Factuur {r['date'].date()}")
139        return etl.addfield(rdr, "payee", "NS Reizigers")

208    @override
209    def custom_entry_mods(self, new_entries: data.Entries):
210        # We now have all parts in seperate transactions. Instead, we want to
211        # combine them all into a single transaction. The transactions have two
212        # postings, the first for `main_account`, the other for the particular
213        # expense.
214
215        def transform_transaction(entry: data.Directive) -> data.Directive:
216            """
217            Do two things:
218            - Add the negative amount to the second posting.
219            - Move the narration to the description of the second posting.
220            """
221            assert isinstance(entry, data.Transaction)  # noqa: S101
222            assert entry.postings[0].units is not None  # noqa: S101
223            assert entry.postings[0].units.number is not None  # noqa: S101
224
225            narration = entry.meta["narration"]
226            del entry.meta["narration"]
227            currency = entry.postings[0].units.currency
228            number = entry.postings[0].units.number
229            entry.postings[1] = entry.postings[1]._replace(
230                units=Amount(-number, currency), meta={"description": narration}
231            )
232            return entry
233
234        new_entries = [transform_transaction(entry) for entry in new_entries]
235
236        if len(new_entries) == 0:
237            return []
238
239        # Copy all postings to the first entry.
240        first_entry = new_entries[0]
241        assert isinstance(first_entry, data.Transaction)  # noqa: S101
242        for other_entry in new_entries[1:]:
243            assert isinstance(other_entry, data.Transaction)  # noqa: S101
244            for posting in other_entry.postings:
245                if posting.meta is not None and posting.meta["description"] is not None:
246                    first_entry.postings.append(posting)
247                else:
248                    # Combine with existing posting
249                    index = None
250                    for i in range(len(first_entry.postings)):
251                        if first_entry.postings[i].account == posting.account:
252                            index = i
253                            break
254                    assert index is not None  # noqa: S101
255                    first_posting = first_entry.postings[index]
256                    assert first_posting.units is not None  # noqa: S101
257                    assert posting.units is not None  # noqa: S101
258                    assert posting.units.number is not None  # noqa: S101
259                    assert first_posting.units.number is not None  # noqa: S101
260                    first_entry.postings[index] = first_posting._replace(
261                        units=first_posting.units._replace(
262                            currency=first_posting.units.currency,
263                            number=first_posting.units.number + posting.units.number,
264                        )
265                    )
266
267        return [first_entry]

A reader that converts a pdf with tables into a multi-petl-table format understood by transaction builders.

Attributes customized in custom_init

self.pdf_table_extraction_settings: {} a dictionary containing settings used to extract tables, see pdfplumber documentation for what settings are available

self.pdf_table_extraction_crop: (int,int,int,int) a tuple with 4 values representing distance from left, top, right, bottom of the page respectively, this will crop the input (each page) before searching for tables

self.pdf_table_title_height: int an integer representing how far up from the top of the table should we look for a table title. Set to 0 to not extract table titles, in which case sections will be labelled as table_# in the order they were encountered

self.pdf_page_break_top: int an integer representing the threshold where a table can be considered page-broken. If the top of a table is lower than the provided value, it will be in consideration for amending to the previous page's last table. Set to 0 to never consider page-broken tables

self.debug: boolean When debug is True a few images and text file are generated: .debug-pdf-metadata-page_#.png shows the text available in self.meta_text with table data blacked out

    .debug-pdf-table-detection-page_#.png
        shows the tables detected with cells outlined in red, and the background light blue. The purple box shows where we are looking for the table title.

    .debug-pdf-data.txt
        is a printout of the meta_text and table data found before being processed into petl tables, as well as some generated helper objects to add to new importers or import configs

self.transaction_table_section: str When reading a pdf that contains transactions, set this setting to the name of the table section that contains the transactions. This is the key for the table in the self.alltables dictionary.

Outputs

self.meta_text: str contains all text found in the document outside of tables

self.alltables: {'table_1': <petl table of first table in document>, ...} contains all the tables found in the document keyed by the extracted title if available, otherwise by the 1-based index in the form of table_#

@override
def custom_entry_mods( self, new_entries: List[Union[beancount.core.data.Open, beancount.core.data.Close, beancount.core.data.Commodity, beancount.core.data.Pad, beancount.core.data.Balance, beancount.core.data.Transaction, beancount.core.data.Note, beancount.core.data.Event, beancount.core.data.Query, beancount.core.data.Price, beancount.core.data.Document, beancount.core.data.Custom]]): 
209    @override
210    def custom_entry_mods(self, new_entries: data.Entries):
211        # We now have all parts in seperate transactions. Instead, we want to
212        # combine them all into a single transaction. The transactions have two
213        # postings, the first for `main_account`, the other for the particular
214        # expense.
215
216        def transform_transaction(entry: data.Directive) -> data.Directive:
217            """
218            Do two things:
219            - Add the negative amount to the second posting.
220            - Move the narration to the description of the second posting.
221            """
222            assert isinstance(entry, data.Transaction)  # noqa: S101
223            assert entry.postings[0].units is not None  # noqa: S101
224            assert entry.postings[0].units.number is not None  # noqa: S101
225
226            narration = entry.meta["narration"]
227            del entry.meta["narration"]
228            currency = entry.postings[0].units.currency
229            number = entry.postings[0].units.number
230            entry.postings[1] = entry.postings[1]._replace(
231                units=Amount(-number, currency), meta={"description": narration}
232            )
233            return entry
234
235        new_entries = [transform_transaction(entry) for entry in new_entries]
236
237        if len(new_entries) == 0:
238            return []
239
240        # Copy all postings to the first entry.
241        first_entry = new_entries[0]
242        assert isinstance(first_entry, data.Transaction)  # noqa: S101
243        for other_entry in new_entries[1:]:
244            assert isinstance(other_entry, data.Transaction)  # noqa: S101
245            for posting in other_entry.postings:
246                if posting.meta is not None and posting.meta["description"] is not None:
247                    first_entry.postings.append(posting)
248                else:
249                    # Combine with existing posting
250                    index = None
251                    for i in range(len(first_entry.postings)):
252                        if first_entry.postings[i].account == posting.account:
253                            index = i
254                            break
255                    assert index is not None  # noqa: S101
256                    first_posting = first_entry.postings[index]
257                    assert first_posting.units is not None  # noqa: S101
258                    assert posting.units is not None  # noqa: S101
259                    assert posting.units.number is not None  # noqa: S101
260                    assert first_posting.units.number is not None  # noqa: S101
261                    first_entry.postings[index] = first_posting._replace(
262                        units=first_posting.units._replace(
263                            currency=first_posting.units.currency,
264                            number=first_posting.units.number + posting.units.number,
265                        )
266                    )
267
268        return [first_entry]
